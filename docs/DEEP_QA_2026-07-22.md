# PhoneDock Deep QA — 2026-07-22

Scope: supplied `phonedock-main (38).zip`, reviewed in an isolated workspace.

## Inventory

- 376 source files
- 71 page routes
- 11 API route entry files with catch-all domain handlers
- 23 model files
- 27 pre-existing regression suites plus deep-QA coverage
- 7 Playwright suites

## Fixed in this pass

1. Price Tracker admin requests no longer silently swallow overview, source, history, settings, toggle or review failures. Non-2xx responses are parsed safely and visible alerts are rendered.
2. Admin phone create/edit pages now report brand-loading failures instead of silently presenting incomplete data.
3. Search retry no longer reloads the browser; it repeats the request in place.
4. JSON-LD uses a shared serializer that escapes script-closing and HTML-significant characters.
5. Duplicate Mongoose index declarations were removed from unique fields while preserving unique indexes.
6. Price-alert cron work is bounded to 500 alerts per invocation and reports whether more work remains.
7. Turnstile callbacks use current callback refs without recreating the widget on every parent render.
8. React callback dependency defects in user/data-quality admin flows were corrected.
9. Empty operational catch blocks in the reviewed source flows were replaced with visible errors; intentional localStorage degradation is documented.

## Route and data review

All 25 literal internal navigation targets found by the static scan map to existing page routes. Dynamic links still require browser/staging validation. Model relationships and indexes were inspected; uniqueness/index duplication was corrected. Several intentional collection-wide maintenance queries remain in data-quality jobs and should only run under controlled operational limits.
